---

## 基础流程
1. 以 task_slug 命名 JSON 文件（如 `acc_bug.json`），禁止使用 `task_forum.json`
2. **每个 step/action 执行前讨论**：PROPOSAL → 调用三审阅者；任一 Agent 意见须被其余 3 个充分审核
3. **意见不一致**：4 方投票，少数服从多数；平票或无法统一时，type: QA 争取我的意见
4. 禁止在未完成当前 step/action 讨论前进入下一步

## 增强协作（必须遵守）
5. **搜索/分析阶段也要共享 context**：发现重要信息立即追加 CONTEXT_SHARE，不等结论
6. **并行子任务**：任务复杂或耗时时，Opus 主动拆分子任务分配给其他 Agent 并行执行
7. 子任务新发现随时 CONTEXT_SHARE 同步，其他 Agent 可 REPLY 补充关联知识
8. 跨任务线索明确 @tag 相关 Agent

## 消息类型
CONTEXT_SHARE | PROPOSAL | REVIEW | SUBTASK_RESULT | REPLY | VOTE | EXECUTION | QA

每次更新后运行：
    python build_forum.py <task_slug>
生成 <task_slug>.html 查看讨论面板。
